# UI / 页面设计

## 1. 设计目标

第 2 阶段只定义 Hotel Scout Agent 的页面信息结构、组件拆分、交互状态与视觉布局，不实现任何前端代码。UI 设计需要服务于 MVP 的三条核心路径：

1. 用户提交酒店比价查询条件。
2. 用户观察平台采集进度，并在登录、验证码、滑块等场景中介入处理。
3. 用户查看带价格、可信度、截图证据和采集时间的结果列表。

## 2. 页面地图

| 页面 | 路由建议 | 主要用户任务 | 设计重点 |
| --- | --- | --- | --- |
| 首页查询页 | `/` | 填写目的地、日期、人数、筛选条件并创建任务 | 表单清晰、校验明确、平台选择显性 |
| 任务执行页 | `/tasks/:taskId` | 查看任务状态、平台进度、人工验证提示 | 状态可观测、问题可定位、操作不误导 |
| 结果列表页 | `/tasks/:taskId/results` | 对比酒店价格、可信度和截图证据 | 最低价突出、证据可追溯、筛选排序明确 |

## 3. 全局布局规范

### 3.1 页面容器

- 桌面端采用居中最大宽度 `1200px` 的内容容器。
- 页面左右保留安全边距，建议桌面端 `24px`，移动端 `16px`。
- 顶部保留统一产品标题区域，展示产品名和当前页面主操作。
- MVP 优先桌面端浏览器体验，移动端只要求内容不溢出。

### 3.2 顶部导航

顶部导航建议包含：

- 产品名：Hotel Scout Agent。
- 当前任务入口：当已有任务时可展示“返回任务”。
- 说明入口：展示“不破解验证码、不保证全网最低、不自动下单”等产品边界提示。

### 3.3 视觉层级

| 层级 | 用途 | 视觉建议 |
| --- | --- | --- |
| 页面标题 | 当前页面目标 | 大字号、粗体、左对齐 |
| 卡片标题 | 查询区域、平台任务、结果分组 | 中字号、粗体 |
| 状态标签 | 运行中、等待人工验证、已完成、失败 | 胶囊标签 + 语义色 |
| 次要说明 | 条件摘要、采集时间、证据说明 | 低对比度文本 |
| 风险提示 | 验证、失败、证据缺失 | 警示色或错误色背景块 |

## 4. 设计令牌草案

> 本节只作为后续前端实现的样式约束，不创建样式代码。

### 4.1 颜色语义

| 语义 | 用途 |
| --- | --- |
| Primary | 主按钮、当前最低价标签、关键聚焦态 |
| Success | 平台完成、可信度高、校验通过 |
| Warning | 等待人工验证、证据缺失、可信度中 |
| Danger | 任务失败、平台失败、日期非法 |
| Neutral | 背景、边框、辅助文本 |

### 4.2 状态色建议

| 状态 | 标签文案 | 语义色 |
| --- | --- | --- |
| `created` | 已创建 | Neutral |
| `running` | 运行中 | Primary |
| `waiting_manual_verification` | 等待人工验证 | Warning |
| `paused` | 已暂停 | Warning |
| `completed` | 已完成 | Success |
| `failed` | 失败 | Danger |
| `cancelled` | 已取消 | Neutral |

### 4.3 间距建议

- 页面区块之间：`24px` - `32px`。
- 卡片内部：`16px` - `24px`。
- 表单字段之间：`12px` - `16px`。
- 按钮组间距：`8px` - `12px`。

## 6. 首页查询页设计摘要

首页采用“左侧主表单 + 右侧说明卡片”的桌面布局。主表单负责创建任务，说明卡片解释采集边界、平台限制和人工验证流程。

核心信息：

- 目的地：城市名称输入框，可展示热门城市建议。
- 入住 / 离店日期：日期范围，必须入住早于离店。
- 人数：MVP 至少支持成人数量。
- 关键词：用于区域、商圈、地标或偏好。
- 价格范围：最低价和最高价。
- 距离筛选：距离市中心、地铁站或指定地点。
- 平台选择：携程、Booking、飞猪、美团，默认可全选或按 MVP 阶段调整。
- 排序方式：价格优先、可信度优先、距离优先。

详细规格见 `specs/001-首页查询页.md`。

## 7. 任务执行页设计摘要

任务执行页采用“顶部任务状态 + 平台卡片网格 + 事件时间线”的结构。页面必须让用户明确知道任务卡在哪一步、是否需要人工介入，以及用户能执行哪些安全操作。

核心信息：

- 任务名称、总状态、查询条件摘要、创建时间。
- 各平台状态、当前步骤、已采集数量、问题说明。
- 人工验证时显示原因、平台、处理步骤和继续按钮。
- 事件时间线展示关键状态变化，帮助定位失败或等待原因。

详细规格见 `specs/002-任务执行页.md`。

## 8. 结果列表页设计摘要

结果列表页采用“结果汇总 + 筛选排序工具栏 + 酒店结果卡片 / 表格”的结构。MVP 可优先卡片列表，后续再补充密集表格视图。

核心信息：

- 酒店名称、位置、平台来源。
- 列表页价格、详情页确认价格、币种。
- 是否最低详情价。
- 可信度等级与降级原因。
- 来源 URL、截图证据、采集时间。
- 无结果、证据缺失、价格不一致等提示。

详细规格见 `specs/003-结果列表页.md`。

## 9. 状态与文案规范

### 9.1 人工验证文案

当系统检测到登录、验证码或滑块：

```txt
检测到 {平台} 需要人工验证。系统不会破解验证码或绕过滑块。
请点击“打开浏览器处理”，在浏览器中完成验证后返回本页点击“我已完成，继续”。
```

### 9.2 价格可信度文案

| 可信度 | 展示文案 | 典型原因 |
| --- | --- | --- |
| 高 | 可信度高 | 有详情页确认价、截图、URL 和采集时间 |
| 中 | 可信度中 | 有列表价或详情价，但部分证据不完整 |
| 低 | 可信度低 | 缺少详情价、缺少截图或页面疑似变化 |

### 9.3 最低价说明

最低价标签只在存在详情页确认价的结果之间比较。没有详情确认价的结果不得标记为最低价。

## 10. 空状态与错误状态

| 场景 | 页面 | 展示策略 |
| --- | --- | --- |
| 表单未填写 | 首页 | 字段下方给出校验说明，禁止创建任务 |
| 无平台选中 | 首页 | 平台选择区展示错误提示 |
| 任务刚创建暂无事件 | 任务页 | 展示“任务已创建，等待调度” |
| 平台等待人工验证 | 任务页 | 高亮人工验证面板，不自动重试 |
| 单个平台失败 | 任务页 | 平台卡片失败，主任务仍可继续其他平台 |
| 无酒店结果 | 结果页 | 说明未获取到可确认价格，不展示最低价 |
| 截图证据缺失 | 结果页 | 证据区域展示缺失提示并降低可信度 |

## 11. 可访问性要求

- 表单字段必须有可见标签和错误提示。
- 状态标签不能只依赖颜色，必须有文字。
- 主要按钮文案必须描述动作，例如“创建比价任务”“我已完成，继续”。
- 证据预览弹窗应支持关闭按钮和键盘关闭。
- 价格差异、可信度和验证等待提示应有清晰文本说明。
