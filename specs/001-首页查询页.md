# 001 - 首页查询页规格

## 1. 页面目标

首页查询页用于收集酒店比价任务的查询条件，并创建采集任务。页面必须让用户明确知道输入内容会被用于多个平台的自动搜索，同时提醒遇到登录、验证码或滑块时需要人工处理。

## 2. 入口与出口

| 类型 | 说明 |
| --- | --- |
| 路由建议 | `/` |
| 主要入口 | 用户打开应用首页 |
| 成功出口 | 创建任务成功后跳转 `/tasks/:taskId` |
| 失败出口 | 停留当前页并展示创建失败原因 |

## 3. 页面布局

桌面端建议分为两列：

```txt
┌──────────────────────────────────────────────┐
│ 顶部：产品名 + 边界说明入口                   │
├──────────────────────────────┬───────────────┤
│ 左侧：查询表单                │ 右侧：说明卡片 │
│ - 目的地                      │ - 工作流程     │
│ - 日期                        │ - 平台限制     │
│ - 人数                        │ - 人工验证说明 │
│ - 关键词 / 价格 / 距离         │               │
│ - 平台 / 排序                 │               │
│ - 创建按钮                    │               │
└──────────────────────────────┴───────────────┘
```

移动端可改为单列，说明卡片置于表单下方。

## 4. 信息结构

### 4.1 主标题区

- 标题：酒店比价采集助手。
- 副标题：输入条件后，系统将模拟人工在平台搜索、核价并保存截图证据。
- 边界提示：不破解验证码、不绕过滑块、不自动下单。

### 4.2 查询表单字段

| 字段 | 类型 | 是否必填 | 默认值 / 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 目的地 | 文本输入 + 热门城市建议 | 是 | 东京 | MVP 不维护完整城市库，优先交给平台联想 |
| 入住日期 | 日期选择 | 是 | 2026-06-01 | 必须早于离店日期 |
| 离店日期 | 日期选择 | 是 | 2026-06-02 | 必须晚于入住日期 |
| 成人数 | 数字步进器 | 是 | 2 | 最小 1 |
| 关键词 | 文本输入 | 否 | 新宿 / 银座 / 近地铁 | 可用于商圈、地标、偏好 |
| 最低价 | 数字输入 | 否 | 0 | 不得小于 0 |
| 最高价 | 数字输入 | 否 | 500 | 若填写必须大于等于最低价 |
| 距离筛选 | 下拉 / 文本输入 | 否 | 距离地铁站近 | 可按市中心、地铁站、指定地点描述 |
| 平台选择 | 多选卡片 | 是 | 携程、Booking、飞猪、美团 | 至少选择 1 个平台 |
| 排序方式 | 单选 | 是 | 价格优先 | 可选价格优先、可信度优先、距离优先 |

### 4.3 平台选择卡片

每个平台卡片展示：

- 平台名称。
- 平台状态说明，例如“可能需要登录验证”。
- 是否选中。

MVP 文案：

| 平台 | 说明 |
| --- | --- |
| 携程 | 中文酒店平台，可能需要登录或滑块验证 |
| Booking | 国际酒店平台，目的地关键词建议使用英文或当地语言 |
| 飞猪 | 中文酒店平台，可能需要登录验证 |
| 美团 | 中文酒店平台，国际目的地覆盖可能有限 |

## 6. 表单校验规则

| 规则 | 失败提示 |
| --- | --- |
| 目的地不能为空 | 请输入目的地城市或关键词 |
| 入住日期不能为空 | 请选择入住日期 |
| 离店日期不能为空 | 请选择离店日期 |
| 入住日期必须早于离店日期 | 离店日期必须晚于入住日期 |
| 成人数必须大于等于 1 | 成人数至少为 1 |
| 最低价不得小于 0 | 最低价不能小于 0 |
| 最高价若填写必须大于等于最低价 | 最高价必须大于等于最低价 |
| 至少选择一个平台 | 请至少选择一个采集平台 |
| 排序方式不能为空 | 请选择排序方式 |

## 7. 交互流程

### 7.1 创建任务成功

1. 用户填写查询条件。
2. 点击“创建比价任务”。
3. 按钮进入 loading 状态，文案为“正在创建任务”。
4. 表单校验通过后提交创建任务请求。
5. 创建成功后跳转到 `/tasks/:taskId`。

### 7.2 创建任务失败

1. 保持用户已填写内容。
2. 在提交按钮上方或页面顶部展示错误提示。
3. 按钮恢复可点击状态。
4. 用户可修改条件后重新提交。

### 7.3 热门城市建议

MVP 可展示固定热门城市建议：东京、上海、北京、广州、深圳、香港、新加坡、曼谷。点击建议后填入目的地输入框。

## 8. 状态设计

| 状态 | 展示 |
| --- | --- |
| 初始 | 表单可编辑，使用默认示例或空值 |
| 校验失败 | 对应字段下方展示错误，创建按钮不提交 |
| 提交中 | 禁用表单主按钮，防止重复提交 |
| 创建失败 | 展示错误提示，保留表单值 |
| 创建成功 | 跳转任务执行页 |

## 9. 文案要求

- 主按钮：创建比价任务。
- 辅助说明：价格结果会受库存、税费、会员价和平台规则影响，仅作为采集时点参考。
- 平台边界：系统不会破解验证码、绕过滑块或伪造登录态。

## 10. 非目标

本页面在第 2 阶段不实现：

- 真实表单代码。
- 城市库接口。
- 任务创建接口。
- 路由跳转代码。
- 平台真实搜索逻辑。
