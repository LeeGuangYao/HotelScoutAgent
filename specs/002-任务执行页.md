# 002 - 任务执行页规格

## 1. 页面目标

任务执行页用于展示酒店采集任务的实时进度、平台子任务状态、人工验证等待点和任务事件。用户应能快速判断任务是否正常运行、哪个平台需要处理、以及处理完成后如何继续。

## 2. 入口与出口

| 类型 | 说明 |
| --- | --- |
| 路由建议 | `/tasks/:taskId` |
| 主要入口 | 首页创建任务成功后跳转 |
| 次要入口 | 从结果页或任务列表返回 |
| 成功出口 | 任务完成后进入 `/tasks/:taskId/results` |
| 异常出口 | 平台失败可留在本页查看原因，任务级失败展示错误状态 |

## 3. 页面布局

```txt
┌──────────────────────────────────────────────┐
│ 顶部：任务名称 + 总状态 + 主要操作             │
├──────────────────────────────────────────────┤
│ 查询条件摘要：目的地 / 日期 / 人数 / 平台       │
├──────────────────────────────────────────────┤
│ 若需要人工验证：人工验证提示面板               │
├──────────────────────────────────────────────┤
│ 平台任务卡片网格                              │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│ │ 携程      │ │ Booking  │ │ 飞猪      │       │
│ └──────────┘ └──────────┘ └──────────┘       │
├──────────────────────────────────────────────┤
│ 任务事件时间线                                │
└──────────────────────────────────────────────┘
```

## 4. 信息结构

### 4.1 任务顶部状态

字段：

- 任务名称，例如“东京酒店比价”。
- 主任务状态：已创建、运行中、等待人工验证、已暂停、已完成、失败、已取消。
- 创建时间、更新时间。
- 已采集酒店数量。
- 主操作按钮：查看结果、暂停、取消、返回首页。

### 4.2 查询条件摘要

展示：

- 目的地。
- 入住日期和离店日期。
- 成人数。
- 关键词。
- 价格范围。
- 距离筛选。
- 平台列表。
- 排序方式。

### 4.3 平台任务卡片

每个平台卡片展示：

| 字段 | 说明 |
| --- | --- |
| 平台名称 | 携程、Booking、飞猪、美团 |
| 平台状态 | pending、opening、searching、collecting_list、confirming_detail、saving_evidence、waiting_manual_verification、skipped、completed、failed |
| 当前步骤 | 面向用户的中文步骤说明 |
| 已采集数量 | 当前平台已采集酒店数量 |
| 当前酒店 | 如果正在进入详情页，可展示酒店名 |
| 问题说明 | 登录、验证码、滑块、页面异常、超时等 |
| 最近更新时间 | 辅助判断是否卡住 |
| 操作按钮 | 打开浏览器处理、我已完成继续、跳过平台 |

## 5. 平台状态展示规则

| 平台状态 | 用户文案 | 操作 |
| --- | --- | --- |
| `pending` | 等待调度 | 无 |
| `opening` | 正在打开页面 | 可暂停任务 |
| `searching` | 正在搜索酒店 | 可暂停任务 |
| `collecting_list` | 正在采集列表页 | 可暂停任务 |
| `confirming_detail` | 正在进入详情页核价 | 可暂停任务 |
| `saving_evidence` | 正在保存截图证据 | 可暂停任务 |
| `waiting_manual_verification` | 等待人工验证 | 打开浏览器处理、我已完成继续、跳过平台 |
| `skipped` | 已跳过 | 无 |
| `completed` | 已完成 | 查看结果 |
| `failed` | 采集失败 | 查看失败原因、可跳过 |

## 6. 人工验证面板

当任一平台处于 `waiting_manual_verification`，页面顶部查询摘要下方展示人工验证面板。

### 6.1 面板内容

- 标题：需要人工验证。
- 平台：例如携程。
- 原因：检测到登录页 / 验证码 / 滑块 / 风控提示。
- 当前步骤：例如“搜索酒店列表前”。
- 当前 URL：可折叠展示，避免页面拥挤。
- 操作说明：系统不会破解验证码，请用户在浏览器中手动完成。
- 操作按钮：打开浏览器处理、我已完成，继续、跳过该平台。

### 6.2 继续流程

1. 用户点击“打开浏览器处理”。
2. 用户在真实浏览器中完成登录或验证。
3. 用户回到任务页点击“我已完成，继续”。
4. 页面按钮进入 loading 状态，文案为“正在恢复”。
5. 如果后端确认验证解除，平台恢复运行。
6. 如果仍需验证，继续保持等待状态并更新提示。

## 7. 事件时间线

事件时间线按时间倒序或正序展示，MVP 建议正序便于理解流程。

事件包含：

- 时间。
- 事件类型。
- 平台名称（如适用）。
- 简短描述。
- 详情展开区（如错误堆栈摘要、URL、恢复点）。

常见事件文案：

| 事件 | 文案 |
| --- | --- |
| `task.created` | 任务已创建 |
| `task.status_changed` | 任务状态变更为 {状态} |
| `platform.status_changed` | {平台} 状态变更为 {状态} |
| `platform.progress` | {平台} 已采集 {count} 家酒店 |
| `manual_verification.required` | {平台} 需要人工验证 |
| `manual_verification.resumed` | 用户已请求继续 {平台} |
| `hotel_result.created` | 新增酒店结果：{酒店名} |
| `evidence.created` | 已保存截图证据 |
| `platform.failed` | {平台} 采集失败 |
| `task.completed` | 任务已完成 |

## 8. 组件拆分

| 组件 | 职责 |
| --- | --- |
| `TaskPage` | 页面容器，加载任务状态与事件 |
| `TaskSummaryHeader` | 任务名、总状态、主操作按钮 |
| `QueryCriteriaSummary` | 查询条件摘要 |
| `ManualVerificationPanel` | 人工验证提示和恢复操作 |
| `PlatformTaskGrid` | 平台卡片布局 |
| `PlatformTaskCard` | 单个平台状态、进度和操作 |
| `TaskControlBar` | 暂停、取消、查看结果等任务级操作 |
| `TaskEventTimeline` | 事件列表展示 |
| `StatusBadge` | 统一状态标签 |

## 9. 空状态与异常状态

| 场景 | 展示策略 |
| --- | --- |
| 任务不存在 | 展示“任务不存在或已被删除”，提供返回首页 |
| 任务已创建但未开始 | 展示“等待调度”，平台卡片为 pending |
| 事件为空 | 展示“暂无事件，任务正在初始化” |
| 单个平台失败 | 平台卡片标红，展示原因，不把主任务直接判定失败 |
| 任务级失败 | 顶部状态标红，说明无法继续，保留事件供排查 |
| 多个平台同时等待验证 | 人工验证面板按平台分组，优先展示最早进入等待的项目 |

## 10. 安全与边界文案

- 不展示“自动处理验证码”之类暗示。
- 对人工验证统一说明：系统只暂停等待用户处理，不破解、不绕过。
- “跳过平台”必须有确认提示，避免用户误操作。

## 11. 非目标

本页面在第 2 阶段不实现：

- SSE 连接代码。
- 任务状态 API。
- 暂停、继续、跳过接口。
- 浏览器窗口控制。
- 实际平台采集逻辑。
